# PLAN — ds-doc-to-decision (pre-approved; recorded for audit)
**Objective.** Ingest 40 messy distributor document sets → extract (PDF + TXT, field-level confidence) → deterministic 3-way match per matching_rules.md (verbatim, the LAW) → decide approve/route/reject with the named failing check → calibrated confidence gate → exception-queue UI with a learning loop that measurably raises STP → robustness + ROI evidence.
**Approach.** Offline stdlib+numpy/pandas/matplotlib/PIL: hand-rolled PDF text extractor (ASCII85+Flate content streams — pdfplumber uninstallable, D1); rules engine as pure functions over the normalized schema; GT-free confidence calibration (target = PDF-path decision agrees with structured-JSON-path decision; GT used ONLY in scoring scripts, enforced by a grep gate per override 4); stdlib HTTP UI; text-domain degradation for robustness (no rasterizer offline, D3).
**Files.** src/{loaders,pdftext,extract,match,decide,pipeline,calibrate,server}.py · ui/index.html · tests/* · EXTRACTION_RESULTS.md · DECISIONS.md · robustness.png · README.md · MORNING_REPORT.md.
**Risks.** PDF operator parsing quirks (mitigate: TXT path is primary evidence; PDF delta reported honestly); 45 s sandbox exec slices (gates run per-call as in prior builds); mount blocks git/sqlite (git separate-dir D2; no DB needed here).
